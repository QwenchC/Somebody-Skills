# 项目开发文档：Somebody-Skills（SBS）

## 1. 项目概述

**项目名称**：Somebody-Skills（简称 SBD）  
**目标**：开发一套完整的本地化“个体蒸馏”工具，能够将任意目标人物（`somebody`）的数字痕迹转化为可部署的 `.skill` 文件。  
**核心优势**：全自动环境配置、多源数据采集、超越 `女娲.skill` 的蒸馏质量、一键封装部署。  
**运行环境**：Node.js 18+ / Python 3.10+，充分利用用户的高性能硬件（例如9950X3D + RTX 5090）进行本地大模型推理与微调。

## 2. 功能需求详细说明

### 2.1 蒸馏准备 —— 自动环境检测与安装

**目标**：用户启动程序后，无需手动配置环境，一键就绪。

**功能点**：
- 检测操作系统（Windows / macOS / Linux）及体系结构。
- 检测并安装/升级以下依赖：
  - Node.js（≥18.0）及 npm/yarn/pnpm
  - Python（≥3.10）及 pip
  - 必需的 Python 包（transformers, torch, unsloth, fastapi, uvicorn, playwright 等）
  - 必需的 Node.js 包（如 `@anthropic-ai/sdk`, `openai`, `puppeteer` 等）
- 检测 GPU 驱动（NVIDIA CUDA 版本 ≥12.8），自动配置 PyTorch 的 CUDA 支持。
- 若用户缺少上述环境，自动静默安装（Windows 下可通过 winget/choco，macOS 通过 brew，Linux 通过包管理器）。需要管理员权限时提示用户授权。
- 生成 `.env` 配置文件模板（包含 API Key 占位符，若需调用云端模型则让用户填写；默认优先使用本地模型）。
- 提供“环境验证”命令（`psd check`），输出详细的健康检查报告。

### 2.2 数据收集 —— 多源数据采集

**目标**：灵活获取目标人物的原始数据，支持自动爬取与手动上传。

**功能点**：
- **模式选择**：
  - 模式 A：自动爬取（用户输入目标名称 + 可选平台列表）
  - 模式 B：手动上传（用户提供本地文件夹或 ZIP 包）
  - 模式 C：混合模式（先自动爬取，再补充本地文件）
- **自动爬取模块**（需遵守 robots.txt 及法律道德）：
  - 支持平台：Twitter/X、微博、GitHub、知乎、微信公众号文章、个人博客（通用爬虫）、LinkedIn（公开信息）、YouTube 字幕（演讲视频）等。
  - 爬取内容：文本（帖子、文章、评论、代码注释）、时间线、对话风格样本。
  - 使用 Playwright/Puppeteer 处理动态页面，支持登录态（用户提供 Cookie 时）。
  - 输出原始数据到 `./workspace/raw/`，按平台分类存储为 JSON/文本文件。
- **手动上传模块**：
  - 支持格式：txt, md, json, csv, pdf（提取文本）, docx, 聊天记录导出（微信/Telegram）。
  - 自动去重、编码检测（UTF-8/GBK）。
- **数据预处理**：
  - 清洗：去除无关 HTML 标签、广告、重复段落。
  - 脱敏：自动识别并模糊化手机号、身份证、邮箱等敏感信息（用户可关闭）。
  - 生成数据摘要报告（文档数量、总字符数、时间跨度、主要话题）。

### 2.3 核心蒸馏 —— 超越女娲.skill

**目标**：提取更精准、更结构化、更具人格化的模型，最终输出中间表示（Persona Representation）。

**优于女娲.skill 的关键设计**：
1. **双轨蒸馏 + 交叉验证**：
   - 轨道 A（工作技能轨）：提取专业知识、技术栈、决策逻辑、工作流。
   - 轨道 B（人格特征轨）：提取语气、幽默感、价值观、情感倾向、社交风格。
   - 交叉验证：两轨互相提问，消除不一致。
2. **多级粒度蒸馏**：
   - 宏观：世界观、核心原则。
   - 中观：领域知识框架、思维模型。
   - 微观：具体话术、常见问答对。
3. **主动学习式蒸馏**：
   - 蒸馏引擎主动生成“探针问题”，让本地大模型（如 Qwen2.5-32B-Instruct）基于已有数据回答，然后与原数据对比，迭代修正人格表示。
4. **多模态扩展**（若数据包含图像/语音）：
   - 提取视觉偏好（如常用表情包、配色风格）。
   - 提取语音特征（语调、节奏，需音频文件）。
5. **可解释性输出**：
   - 生成一份 `persona_report.html`，可视化展示提炼出的各个维度，并附带原始证据片段（高亮显示）。
6. **性能优化**（利用 5090）：
   - 支持 4-bit/8-bit 量化推理。
   - 使用 Flash Attention 2 加速长文本处理。
   - 支持多卡并行（若有多 GPU）。

**蒸馏流程**：
1. **加载数据**：从 `./workspace/raw/` 读取所有文本。
2. **分块与嵌入**：按 2048 token 分块，使用本地 embedding 模型（如 BAAI/bge-large-zh-v1.5）向量化。
3. **主题聚类**：用 HDBSCAN 聚类，识别主要话题（技术、生活、工作等）。
4. **人格原语提取**：
   - 调用本地 LLM（默认 `Qwen2.5-32B-Instruct-4bit`）对每个聚类生成“人格陈述”。
   - 例如：“该人物在讨论代码规范时，倾向于使用具体例子而非抽象原则。”
5. **冲突检测**：同一维度出现相反陈述时，通过时间戳权重或用户交互（让用户选择）解决。
6. **生成结构化人格 JSON**：格式符合 `SKILL.md` 所需的指令规范。

### 2.4 封装与部署 —— 生成 .skill 文件

**目标**：将蒸馏结果转化为符合主流 AI 编程工具（Claude Code、Cursor 等）标准的 `.skill` 文件夹。

**功能点**：
- **生成 SKILL.md**：
  - 自动填充 YAML Frontmatter：`name`, `description`, `version`, `author`, `created_at`。
  - `description` 由蒸馏引擎自动生成一句话总结。
  - `instruction` 部分：采用 Markdown 格式，按照“角色设定→知识库→行为准则→输出格式”组织。
- **可选附件**：
  - `references/`：存放原始数据精华摘要（向量索引）。
  - `scripts/`：若需要运行时调用某些工具（如搜索、计算），可打包 Python/Node 脚本。
- **打包**：输出为 `{persona_name}.skill.zip` 或直接生成文件夹。
- **安装辅助**：提供命令 `psd install ./output/somebody.skill`，自动复制到目标工具的 `skills/` 目录（支持 Cursor、Claude Code、Continue 等）。
- **测试对话**：生成后自动启动一个本地对话界面（基于 Gradio 或 TUI），让用户与生成的 skill 交互测试，反馈质量评分。

## 3. 系统架构

```text
[CLI / Web UI]
     │
     ▼
┌─────────────────────────────────────────┐
│            Orchestrator                  │
│  (调度各模块，管理工作流状态)             │
└─────────────────────────────────────────┘
     │
     ├─── Module 1: Env Manager ──> 系统检测、自动安装
     ├─── Module 2: Data Collector ──> 爬虫 + 上传接口
     ├─── Module 3: Distillation Engine ──> LLM + Embedding + Clustering
     └─── Module 4: Packager ──> 生成 .skill + 部署
```

- **Orchestrator**：基于有限状态机实现，支持断点续跑（例如爬取中断后可恢复）。
- **数据存储**：SQLite 记录中间状态（文件 hash、蒸馏迭代版本）。
- **本地模型服务**：启动一个常驻的 LLM 服务（使用 llama.cpp 或 vLLM），所有模块通过 HTTP/gRPC 调用，避免重复加载模型。

## 5. 模块详细设计

### 5.1 环境管理器 (EnvManager)
- 入口：`psd setup`
- 实现策略：
  1. 运行 `node -v`，若不存在则调用包管理器安装 Node LTS。
  2. 运行 `python --version`，若不存在则提示用户安装或自动下载 miniconda。
  3. 创建虚拟环境 `.venv`，安装 `torch` (CUDA 版)、`transformers` 等。
  4. 使用 `npx` 安装必要的 Node 全局工具。
- 配置文件：`~/.psd/config.json` 存储环境路径。

### 5.2 数据采集器 (DataCollector)
- 爬虫配置：用户需提供目标名称（如“张三”），可选指定平台列表。
- 针对每个平台实现插件化架构（`src/crawlers/`）。
- 反爬策略：随机 User-Agent、延迟、使用代理池（可选）。
- 手动上传：通过 `psd upload --path ./my_data` 复制文件到工作区。

### 5.3 蒸馏引擎 (DistillationEngine) —— 核心创新区
- **优于女娲.skill 的具体实现**：
  1. **主动探针**：预定义一组 200 个通用探针问题（如“当遇到技术分歧时，你会怎么做？”），让模型基于数据模拟回答，再将回答聚类形成人格向量。
  2. **对比学习**：随机抽取非目标人物的文本作为负样本，增强人格特征的区分度。
  3. **多轮辩论**：两个相同配置的 LLM 实例分别从“工作轨”和“人格轨”出发，互相质疑，最终达成一致。
  4. **用户反馈回路**：蒸馏过程中，抽选 10 个关键陈述让用户打分（“非常同意”到“完全不同意”），动态调整权重。
- **性能优化**：利用 5090 的 FP8 支持，使用 `torch.compile` 加速。

### 5.4 封装器 (Packager)
- 读取蒸馏输出的 `persona.json`，按照模板渲染 `SKILL.md`。
- 模板示例：
  ```markdown
  ---
  name: {{name}}
  description: {{description}}
  version: 1.0.0
  ---
  # 角色设定
  你是 {{name}} 的数字分身，拥有其知识、风格与价值观。

  ## 知识库
  {{#each knowledge_base}}
  - {{this}}
  {{/each}}

  ## 行为准则
  1. 回答时优先使用短句，偶尔使用表情符号 😄。
  2. 对于不确定的问题，回答“我需要查一下资料”。

  ## 输出格式
  ...
  ```
- 同时生成 `persona.db`（向量库）供 skill 运行时检索。

## 7. 优于女娲.skill 的量化指标

| 指标                 | 女娲.skill | PSD 目标    | 实现方式                          |
| -------------------- | ---------- | ----------- | --------------------------------- |
| 人格维度数量         | ~12        | ≥20         | 双轨+交叉验证                     |
| 生成一致性（重复生成） | 70%        | ≥85%        | 主动探针+用户反馈                 |
| 长文本支持（>100K）  | 否         | 是          | 滑动窗口+摘要递归                 |
| 多模态支持           | 文本       | 文本+图像+语音 | 调用 CLIP / Whisper 提取特征    |
| 蒸馏速度（100MB 数据） | 30 分钟    | 15 分钟      | 利用 5090 + Flash Attention 2     |

## 8. 开发计划（建议 6 周）

- **第 1 周**：搭建基础框架（CLI 脚手架、配置管理、日志系统），完成环境检测模块。
- **第 2 周**：实现数据采集模块（至少支持 GitHub + 本地文件），完成爬虫基础。
- **第 3 周**：实现蒸馏引擎原型（单轨，使用本地 Qwen-7B），验证流程。
- **第 4 周**：实现双轨蒸馏、主动探针、辩论机制，优化 prompt。
- **第 5 周**：封装模块开发，生成标准 .skill 文件，实现部署与测试对话。
- **第 6 周**：集成测试、性能调优、撰写用户文档。

## 9. 注意事项

- **法律与伦理**：爬取公开数据时需遵守目标网站的 robots.txt 及服务条款；蒸馏他人需获得明确授权；工具内置免责声明与知情同意流程。
- **隐私保护**：所有数据默认本地处理，不上传云端；生成的 .skill 文件若分享需用户自行负责。
- **健壮性**：任何模块失败应有清晰的错误提示和恢复建议（如“请手动安装 Node.js 18+”）。
- **用户体验**：提供详细的进度条（`cli-progress`），关键步骤输出示例。
